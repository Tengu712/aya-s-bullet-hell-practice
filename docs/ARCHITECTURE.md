# ARCHITECTURE

## Engine

本プロジェクトにおいて、エンジンは画像描画、音楽再生、入力検知、それらに関連する機能を有する。
逆に、シーン遷移、オブジェクト管理、データ管理等々のゲームに関連する機能を有さない。

また、動的リンクライブラリとして実装・配布される。
これは、ユーザに対して再利用性を提供するためである。

次の理由により、主にZigで書かれている。

- C/C++で書きたくなかったため
- ZigがCのヘッダーファイルを読めるため
- ZigにGCがないため
- Zigで書きたかったため

描画APIにVulkanを用いている。
ZigでDirect3Dを叩くのは面倒であるため、たといクロスプラットフォームを視野に入れなくとも妥当の選択である。

## Client

本プロジェクトにおいて、ゲーム部分をクライアントと呼ぶ。
エンジンが動的リンクライブラリとして分離している都合上、ユーザは新しくクライアントを作成・置換できる。

本ソースコードにおいては、.NET C#で書かれている。
オブジェクト指向で設計したかったためである。
